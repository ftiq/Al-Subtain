## مقدمة

نظام **إدارة الوثائق والعمليات (MGT Documents)** هو نظام متكامل لإدارة دورة حياة الوثائق الكاملة داخل المؤسسة، من الإنشاء وحتى الأرشفة، مع دعم كامل لعمليات الموافقة الهرمية، سير العمل الآلي، والتوجيه الذكي للوثائق.

### الميزات الرئيسية:
- إدارة شاملة للوثائق (واردة، صادرة، داخلية، تعاميم)
- نظام موافقات هرمي ذكي
- سير عمل قابل للتخصيص
- توجيه تلقائي للوثائق
- إدارة إصدارات الوثائق
- سجل تغييرات كامل
- تقارير وتحليلات متقدمة
- نظام مهام إدارية
- أرشفة تلقائية

---

## صلاحيات المستخدمين والمجموعات

يعتمد النظام على 6 مجموعات صلاحيات رئيسية مبنية على نموذج RBAC (التحكم بالوصول بناءً على الأدوار):

### 1. عارض وثائق (Document Viewer)
**المعرّف التقني**: `group_doc_viewer`

**الصلاحيات:**
- قراءة الوثائق ضمن نطاق القسم والشركة
- عرض تفاصيل الوثائق
- قراءة المرفقات
- عرض سجل التغييرات (للقراءة فقط)

**القيود:**
- لا يمكن إنشاء وثائق جديدة
- لا يمكن تعديل الوثائق
- لا يمكن حذف الوثائق
- لا يمكن اعتماد أو رفض الوثائق

**الوصول للوثائق:**
- الوثائق التي أنشأها المستخدم نفسه
- الوثائق المحالة إليه كمستلم
- وثائق قسمه (حسب قواعد الوصول)

---

### 2. مُساهِم وثائق (Document Contributor)
**المعرّف التقني**: `group_doc_contributor`

**يرث صلاحيات**: عارض وثائق

**الصلاحيات الإضافية:**
- إنشاء وثائق جديدة
- تعديل الوثائق التي أنشأها
- تعديل الوثائق المحالة إليه
- إرسال وثائق للموافقة
- إنشاء مهام إدارية
- رفع مرفقات
- إضافة ملاحظات وتعليقات

**القيود:**
- لا يمكن اعتماد الوثائق
- لا يمكن حذف وثائق الآخرين
- لا يمكن الوصول للإعدادات
- لا يمكن أرشفة الوثائق

**الوصول للوثائق:**
- جميع صلاحيات العارض
- تعديل الوثائق التي أنشأها
- تعديل الوثائق المحالة إليه كموظف

---

### 3. معتمد وثائق (Document Approver)
**المعرّف التقني**: `group_doc_approver`

**يرث صلاحيات**: عارض وثائق

**الصلاحيات الإضافية:**
- اعتماد طلبات الموافقة المسندة إليه
- رفض طلبات الموافقة
- إضافة تعليقات على الموافقات
- تصعيد طلبات الموافقة
- إنشاء مهام بعد الموافقة (عبر معالج خاص)
- بدء مراجعة الوثائق
- اعتماد الوثائق
- رفض الوثائق
- تعليق معالجة الوثائق

**القيود:**
- يمكنه فقط الموافقة على الطلبات المسندة إليه
- لا يمكن تعديل طلبات موافقة الآخرين
- لا يمكن الوصول للإعدادات العامة

**الوصول للموافقات:**
- طلبات الموافقة المخصصة له كموافق
- طلبات الموافقة لوثائق قسمه (للقراءة)

---

### 4. مدير قسم الوثائق (Document Manager)
**المعرّف التقني**: `group_doc_manager`

**يرث صلاحيات**: مُساهِم وثائق + معتمد وثائق

**الصلاحيات الإضافية:**
- إدارة وثائق القسم والأقسام التابعة
- بدء تنفيذ الوثائق
- إكمال معالجة الوثائق
- استئناف الوثائق المعلقة
- إلغاء إلغاء/رفض الوثائق
- إرجاع الوثائق لمسودة
- أرشفة الوثائق المكتملة
- إدارة فئات الوثائق
- إدارة قواعد التوجيه
- إدارة عمليات سير العمل
- عرض سجلات التغييرات
- عرض إدارة الأرشيف

**القيود:**
- محدود بوثائق قسمه والأقسام التابعة
- لا يمكن الوصول لبعض الإعدادات المتقدمة

**الوصول للوثائق:**
- جميع وثائق قسمه
- وثائق الأقسام التابعة لقسمه
- الوثائق السرية للغاية فقط إذا كان المُنشئ أو المستلم

---

### 5. مسؤول السجلات والأرشيف (Records Manager)
**المعرّف التقني**: `group_doc_records`

**يرث صلاحيات**: مُساهِم وثائق

**الصلاحيات الإضافية:**
- إدارة الأرشيف الكامل
- أرشفة الوثائق
- استرجاع الوثائق من الأرشيف
- تعديل بيانات الأرشفة
- إدارة مواقع التخزين

**التركيز الوظيفي:**
هذا الدور متخصص في عمليات الحفظ والأرشفة والاسترجاع، ويتعامل مع الوثائق المكتملة والمؤرشفة بشكل أساسي.

---

### 6. مسؤول نظام الوثائق (Documents Admin)
**المعرّف التقني**: `group_doc_admin`

**يرث صلاحيات**: مدير قسم الوثائق + صلاحيات النظام

**الصلاحيات الكاملة:**
- الوصول الكامل لجميع الوثائق في النظام
- إدارة جميع المستخدمين والصلاحيات
- تعديل جميع الإعدادات
- حذف أي وثيقة
- إدارة الفئات والتصنيفات
- إدارة قواعد التوجيه
- إدارة عمليات سير العمل
- الوصول لجميع التقارير
- إدارة المهام الإدارية
- إدارة طلبات الموافقة
- التحكم الكامل بالنظام

**بدون قيود**: هذا الدور لديه صلاحيات كاملة ضمن النظام.

## نظام الوثائق

### نموذج البيانات: `document.document`

الوثيقة هي العنصر الأساسي في النظام، وتحتوي على الحقول والميزات التالية:

#### 1. المعلومات الأساسية

**الحقول:**
- **name** (العنوان): نص مطلوب، عنوان الوثيقة
- **reference_number** (الرقم المرجعي): يُنشأ تلقائياً حسب نوع الوثيقة واتجاهها
- **date** (التاريخ): تاريخ الوثيقة (افتراضي: تاريخ اليوم)
- **subject** (الموضوع): موضوع تفصيلي للوثيقة
- **summary** (الملخص): ملخص نصي للوثيقة
- **content** (المحتوى): محتوى HTML كامل للوثيقة

#### 2. التصنيف والنوع

**document_direction** (اتجاه الوثيقة):
- `incoming`: وارد
- `outgoing`: صادر
- `internal`: داخلي

**document_type** (نوع الوثيقة):
- `official_letter`: رسالة رسمية
- `memo`: مذكرة
- `report`: تقرير
- `circular`: تعميم
- `decision`: قرار
- `contract`: عقد
- `invoice`: فاتورة
- `permit`: تصريح
- `request`: طلب
- `other`: أخرى

**category_id**: فئة الوثيقة (Many2one → document.category)

#### 3. الأطراف

**للوثائق الواردة (incoming):**
- **sender_id** (المرسل): جهة خارجية (Many2one → res.partner) - مطلوب
- **recipient_employee_id** (المستلم الموظف): موظف داخلي (Many2one → hr.employee) - مطلوب

**للوثائق الصادرة (outgoing):**
- **sender_employee_id** (المرسل الموظف): موظف داخلي (Many2one → hr.employee) - مطلوب
- **recipient_id** (المستلم): جهة خارجية (Many2one → res.partner) - مطلوب

**للوثائق الداخلية (internal):**
- **sender_employee_id** (المرسل الموظف): موظف داخلي (Many2one → hr.employee) - مطلوب
- **recipient_employee_id** (المستلم الموظف): موظف داخلي (Many2one → hr.employee) - مطلوب

**department_id** (القسم): القسم المعني (Many2one → hr.department) - مطلوب للصادرة والداخلية

#### 4. حالات سير العمل

**workflow_state** (حالة سير العمل):
- `draft`: مسودة (الحالة الافتراضية)
- `submitted`: مرسلة للمعالجة
- `in_review`: قيد المراجعة
- `pending_approval`: في انتظار الموافقة
- `in_progress`: قيد التنفيذ
- `approved`: معتمدة
- `completed`: مكتملة
- `archived`: مؤرشفة
- `cancelled`: ملغاة

**processing_state** (حالة المعالجة):
- `received`: مستلمة (الحالة الافتراضية)
- `registered`: مسجلة
- `routed`: موجهة
- `under_review`: قيد المراجعة
- `approved`: معتمدة
- `in_execution`: قيد التنفيذ
- `completed`: مكتملة
- `rejected`: مرفوضة
- `on_hold`: معلقة
- `archived`: مؤرشفة

#### 5. الأولوية والسرية

**priority** (الأولوية):
- `0`: منخفضة (افتراضي)
- `1`: عادية
- `2`: مرتفعة
- `3`: عاجلة

**confidentiality_level** (مستوى السرية):
- `normal`: عادي (افتراضي)
- `confidential`: سري
- `high`: سري للغاية (Top Secret) - وصول محدود جداً

**urgency_level** (مستوى الاستعجال):
- `normal`: عادي (افتراضي)
- `urgent`: عاجل
- `very_urgent`: عاجل جداً

#### 6. سير العمل والمهام

- **workflow_id**: عملية سير العمل المرتبطة (Many2one → document.workflow.instance)
- **current_step**: الخطوة الحالية في سير العمل (Char)
- **task_ids**: المهام المرتبطة (One2many → admin.task)
- **approval_requests**: طلبات الموافقة (One2many → approval.request)

#### 7. التواريخ والمواعيد

- **received_date**: تاريخ الاستلام (تلقائي عند الإنشاء)
- **due_date**: تاريخ الاستحقاق (مطلوب للصادرة والداخلية)
- **target_completion_date**: التاريخ المستهدف للإنجاز (محسوب من SLA)
- **actual_completion_date**: تاريخ الإنجاز الفعلي
- **processing_duration**: مدة المعالجة بالساعات (محسوب)
- **is_overdue**: هل متأخرة؟ (Boolean محسوب)

#### 8. الإصدارات والتاريخ

- **version_ids**: الإصدارات (One2many → document.version)
- **version_count**: عدد الإصدارات (محسوب)
- **history_ids**: سجل التغييرات (One2many → document.history)
- **history_count**: عدد التغييرات (محسوب)

#### 9. الوثائق المرتبطة

- **related_documents**: وثائق مرتبطة (Many2many → document.document)
- **related_documents_count**: عدد الوثائق المرتبطة (محسوب)
- **keywords**: كلمات مفتاحية (Char)

#### 10. الأرشفة والاحتفاظ

- **is_archived**: مؤرشفة؟ (Boolean)
- **retention_period**: فترة الاحتفاظ بالأشهر (Integer)
- **notes**: ملاحظات داخلية (Text)
- **handling_instructions**: تعليمات المعالجة (Text)
- **escalation_notes**: ملاحظات التصعيد (Text)

### دورة حياة الوثيقة - بالتفصيل الكامل

#### 1. الإنشاء (Creation)

**الخطوات:**
1. اذهب إلى: **إدارة الوثائق → الوثائق → إنشاء**
2. املأ الحقول المطلوبة:
   - العنوان (name)
   - نوع الوثيقة (document_type)
   - اتجاه الوثيقة (document_direction)
   - الفئة (category_id)
   - المرسل/المستلم حسب الاتجاه
   - القسم (للصادرة والداخلية)

**عند الحفظ:**
- يُنشأ **reference_number** تلقائياً حسب نوع الوثيقة واتجاهها
- تُضبط **workflow_state** على `draft`
- تُضبط **processing_state** على `received`
- يُنشأ سجل في **document.history** بالحدث `created`
- إذا كانت الفئة تتطلب موافقة (**category_id.requires_approval = True**)، يُضبط **needs_approval = True**
- إذا كان هناك **workflow تلقائي** مرتبط بالفئة أو النوع، يتم تفعيله تلقائياً

**الرقم المرجعي (reference_number):**
يُنشأ من تسلسلات (ir.sequence) محددة حسب:
- `document.incoming.official_letter` → `INC/OFL/XXXX/2025`
- `document.incoming.memorandum` → `INC/MEM/XXXX/2025`
- `document.outgoing.official_letter` → `OUT/OFL/XXXX/2025`
- `document.outgoing.circular` → `OUT/CIR/XXXX/2025`
- `document.internal.memorandum` → `INT/MEM/XXXX/2025`
- إلخ...

#### 2. التسجيل (Registration)

**الزر:** `action_register`

**الشروط لظهور الزر:**
```python
invisible="document_direction != 'incoming' or processing_state not in ['received'] or workflow_state != 'draft'"
groups="mgt_documents.group_doc_contributor,mgt_documents.group_doc_approver"
```

**من يمكنه:**
- مُساهِم وثائق
- معتمد وثائق
- المدراء والمسؤولون

**ماذا يحدث:**
- تتغير **processing_state** من `received` إلى `registered`
- يُسجل الحدث في **document.history**
- تصبح الوثيقة رسمية في النظام

**الكود الفعلي:**
```python
def action_register(self):
    self.ensure_one()
    if self.processing_state == 'received':
        self.processing_state = 'registered'
        self._log_history('registered', 'تم تسجيل الوثيقة رسمياً')
    return True
```

#### 3. التوجيه (Routing)

**الزر:** `action_route`

**الشروط لظهور الزر:**
```python
invisible="document_direction != 'incoming' or processing_state not in ['registered','received']"
groups="mgt_documents.group_doc_approver,mgt_documents.group_doc_manager"
```

**من يمكنه:**
- معتمد وثائق
- مدير قسم
- مسؤول النظام

**ماذا يحدث:**
1. يبحث النظام عن **قواعد توجيه نشطة** (document.router حيث state='active')
2. يفلتر القواعد حسب:
   - نوع الوثيقة (filter_document_type)
   - الأولوية (filter_priority)
   - الأقسام (filter_departments)
   - الفئات (applicable_category_ids)
   - الكلمات المفتاحية (filter_keywords)
3. **لكل قاعدة مطابقة:**
   - إذا كان `target_process_id` محدداً: يُنشئ نسخة من سير العمل
   - إذا كان `create_approval_request=True`: يُنشئ طلب موافقة
   - إذا كان `task_template` محدداً: يُنشئ مهمة
   - إذا كان `notification_message` محدداً: يُرسل إشعار
4. تتغير **processing_state** إلى `routed`
5. يُسجل الحدث في **document.history**

**مثال على الكود:**
```python
def action_route(self):
    self.ensure_one()
    router_model = self.env['document.router']
    applicable_rules = router_model.search([('state', '=', 'active')], order='priority desc, sequence asc')
    
    results = []
    for rule in applicable_rules:
        if self._matches_router_rule(rule):
            result = rule.process_document(self)
            results.append(result)
    
    if results:
        self.processing_state = 'routed'
        self._log_history('routed', f'تم توجيه الوثيقة وفق {len(results)} قاعدة')
    
    return True
```


**من يمكنه:**
- مدير قسم
- مسؤول النظام

**ماذا يحدث:**
- تتغير **workflow_state** إلى `draft`
- تتغير **processing_state** إلى `received`
- تُلغى جميع المهام المرتبطة غير المنتهية
- يُفك ارتباط سير العمل
- تُلغى طلبات الموافقة المعلقة

**الكود الفعلي:**
```python
def action_reset_to_draft(self):
    self.ensure_one()
    # Cancel pending tasks
    pending_tasks = self.task_ids.filtered(lambda t: t.state not in ['completed', 'cancelled'])
    pending_tasks.write({'state': 'cancelled'})
    
    # Unlink workflow
    if self.workflow_id:
        self.workflow_id.action_cancel()
        self.workflow_id = False
    
    # Cancel pending approvals
    pending_approvals = self.approval_requests.filtered(lambda a: a.request_status not in ['approved', 'rejected', 'cancelled'])
    pending_approvals.write({'request_status': 'cancelled'})
    
    # Reset states
    self.workflow_state = 'draft'
    self.processing_state = 'received'
    self._log_history('reset_to_draft', 'تم إرجاع الوثيقة للمسودة')
    
    return True
```

2. **يبني سلسلة موافقات هرمية** باستخدام:

   ```python
   def _build_hierarchical_approvers(self):
       approvers = []
       current_emp = self.sender_employee_id
       
       # إضافة مدير القسم المباشر
       if current_emp.parent_id:
           approvers.append({
               'employee_id': current_emp.parent_id.id,
               'user_id': current_emp.parent_id.user_id.id,
               'sequence': 1,
               'status': 'new'
           })
       
       # إضافة المدراء في السلسلة الهرمية
       level = 2
       next_manager = current_emp.parent_id.parent_id if current_emp.parent_id else False
       while next_manager and level <= 5:  # حتى 5 مستويات
           if self._can_approve_confidentiality_level(next_manager):
               approvers.append({
                   'employee_id': next_manager.id,
                   'user_id': next_manager.user_id.id,
                   'sequence': level,
                   'status': 'new'
               })
               level += 1
           next_manager = next_manager.parent_id
       
       return approvers
   ```
3. يُنشئ **approval.request** مع سلسلة الموافقات
4. تتغير **workflow_state** إلى `pending_approval`

```python
def calculate_due_date(self, start_date=None, priority=False, department=False):
    """
    يحسب تاريخ الاستحقاق بناءً على SLA وأيام العمل
    """
    sla_hours = self.get_sla_hours(priority, department)
    business_days_needed = math.ceil(sla_hours / 8)  # 8 ساعات عمل في اليوم
    due_date = self._add_working_days(start_date or fields.Datetime.now(), business_days_needed)
    return due_date
```

#### 3. إضافة أيام عمل:
```python
def _add_working_days(self, start_date, days):
    """
    يضيف أيام عمل (يتخطى عطلات نهاية الأسبوع: الجمعة والسبت)
    """
    current = start_date
    added = 0
    while added < days:
        current += timedelta(days=1)
        # تخطي الجمعة (4) والسبت (5)
        if current.weekday() not in [4, 5]:
            added += 1
    return current
```

#### 4. تاريخ التصعيد:
```python
def get_escalation_date(self, start_date=None, priority=False):
    """
    يحسب تاريخ التصعيد (75% من وقت SLA)
    """
    sla_hours = self.get_sla_hours(priority)
    escalation_hours = sla_hours * 0.75
    escalation_date = (start_date or fields.Datetime.now()) + timedelta(hours=escalation_hours)
    return escalation_date
```

#### 5. تواريخ الإشعارات:
```python
def get_notification_dates(self, start_date=None, priority=False):
    """
    يعيد قائمة بتواريخ الإشعارات:
    - 50% من الوقت
    - 75% من الوقت (التصعيد)
    - 90% من الوقت
    """
    sla_hours = self.get_sla_hours(priority)
    start = start_date or fields.Datetime.now()
    
    return {
        'first_reminder': start + timedelta(hours=sla_hours * 0.5),
        'escalation': start + timedelta(hours=sla_hours * 0.75),
        'final_reminder': start + timedelta(hours=sla_hours * 0.9)
    }
```

```python
def is_accessible_by_user(self, user=None):
    """
    يفحص إذا كان المستخدم يمكنه الوصول لهذه الفئة
    """
    if not user:
        user = self.env.user
    
    # المسؤول يمكنه الوصول لكل شيء
    if user.has_group('mgt_documents.group_doc_admin'):
        return True

    return True
```

#### 3. بناء سلسلة الموافقات الهرمية:

**الكود الكامل:**
```python
def _build_hierarchical_approvers(self):
    """
    يبني سلسلة موافقات هرمية بناءً على:
    1. التسلسل الهرمي للموظفين (hr.employee.parent_id)
    2. مستوى السرية
    3. حتى 5 مستويات
    """
    self.ensure_one()
    approvers = []
    
    # ابدأ من الموظف المُرسل
    current_emp = self.sender_employee_id or (self.env.user.employee_id if self.env.user.employee_id else False)
    
    if not current_emp:
        return approvers
    
    # المدير المباشر (المستوى 1)
    if current_emp.parent_id and current_emp.parent_id.user_id:
        approvers.append({
            'employee_id': current_emp.parent_id.id,
            'user_id': current_emp.parent_id.user_id.id,
            'sequence': 1,
            'status': 'pending',  # أول موافق يكون 'pending'، الباقي 'new'
            'expected_approval_date': fields.Datetime.now() + timedelta(days=2)
        })
        current_emp = current_emp.parent_id
    
    # المدراء في السلسلة الهرمية (حتى 5 مستويات)
    level = 2
    while current_emp and current_emp.parent_id and level <= 5:
        next_manager = current_emp.parent_id
        
        # تحقق من صلاحية الموافقة حسب مستوى السرية
        if self._can_approve_confidentiality_level(next_manager):
            if next_manager.user_id:
                approvers.append({
                    'employee_id': next_manager.id,
                    'user_id': next_manager.user_id.id,
                    'sequence': level,
                    'status': 'new',  # سيتغير لـ 'pending' عند دوره
                    'expected_approval_date': fields.Datetime.now() + timedelta(days=2*level)
                })
                level += 1
        
        current_emp = next_manager
    
    return approvers

def _can_approve_confidentiality_level(self, employee):
    """
    يتحقق إذا كان الموظف يمكنه الموافقة على مستوى السرية هذا
    """
    if self.confidentiality_level == 'high':
        # السري للغاية: فقط المدراء من المستوى 3 وما فوق
        # (يمكن تخصيص هذا حسب الحاجة)
        return employee.job_id and 'مدير' in (employee.job_id.name or '')
    return True
```

```python
    """
    1. يوافق على الطلب
    2. يُنشئ المهمة
    3. يُرسل الإشعارات

    self.ensure_one()

            'related_task_ids': [(4, task.id)]
        })
        
        # إرسال الإشعار

        # إضافة رسالة في الوثيقة

def _prepare_task_values(self):
    """تحضير قيم المهمة"""
    description = self.task_description or ''

        'legal_procedure': 'other',
        'financial_procedure': 'other',
        'administrative_procedure': 'other',
        'other': 'other',
    }
    
    return {
        'name': self.task_title,
        'description': description,
        'task_type': type_map.get(self.task_type, 'other'),
```

```python
def send_due_task_reminders(self):
    """
    يُنفذ يومياً: يرسل تذكير بالمهام المتأخرة والمهام القريبة من الاستحقاق
    """
    today = fields.Datetime.now()
    tomorrow = today + timedelta(days=1)
    
    # المهام المتأخرة
    overdue_tasks = self.search([
        ('state', 'in', ['assigned', 'in_progress']),
        ('due_date', '<', today)
    ])
    
    for task in overdue_tasks:
        if task.assigned_employee_id and task.assigned_employee_id.user_id:
            task.message_post(
                body=f"<p><strong>تذكير:</strong> هذه المهمة متأخرة!</p>"
                     f"<p>تاريخ الاستحقاق: {task.due_date}</p>",
                partner_ids=[task.assigned_employee_id.user_id.partner_id.id],
                message_type='notification'
            )
    
    # المهام القريبة من الاستحقاق (خلال 24 ساعة)
    upcoming_tasks = self.search([
        ('state', 'in', ['assigned', 'in_progress']),
        ('due_date', '>=', today),
        ('due_date', '<=', tomorrow)
    ])
    
    for task in upcoming_tasks:
        if task.assigned_employee_id and task.assigned_employee_id.user_id:
            task.message_post(
                body=f"<p><strong>تذكير:</strong> موعد استحقاق هذه المهمة غداً!</p>"
                     f"<p>تاريخ الاستحقاق: {task.due_date}</p>",
```

### نموذج البيانات: `workflow.step`

#### الحقول الأساسية:

1. **name**: اسم الخطوة (Char, مطلوب)
2. **process_id**: العملية (Many2one → workflow.process, مطلوب)
3. **sequence**: الترتيب (Integer, default=10)
4. **description**: الوصف (Text)

#### نوع الخطوة:

**step_type** (نوع الخطوة):
- `manual`: يدوية (تتطلب تدخل بشري)
- `automatic`: تلقائية (تُنفذ تلقائياً)
- `approval`: موافقة (تتطلب موافقة)
- `notification`: إشعار (فقط إرسال إشعار)

#### المسؤول:

**responsible_type** (نوع المسؤول):
- `specific_user`: مستخدم محدد
- `department`: قسم
- `role`: دور وظيفي
- `document_owner`: مالك الوثيقة

- **responsible_user_id**: المستخدم المسؤول (Many2one → res.users)
- **responsible_department_id**: القسم المسؤول (Many2one → hr.department)
- **responsible_role**: الدور المسؤول (Char)

#### التوقيتات:

- **estimated_hours**: الساعات المقدرة (Float)
- **deadline_hours**: ساعات الموعد النهائي (Float)

#### الشروط:

- **required_conditions**: الشروط المطلوبة (Text)
- **skip_conditions**: شروط التخطي (Text)
- **is_mandatory**: إلزامية؟ (Boolean, default=True)

#### الموافقة:

- **requires_approval**: تتطلب موافقة؟ (Boolean)

#### الإشعارات:

- **send_notification**: إرسال إشعار؟ (Boolean)
- **notification_template_id**: قالب الإشعار (Many2one → mail.template)

#### الخطوات التالية:

- **next_step_ids**: الخطوات التالية (Many2many → workflow.step)

#### مُحسوبة:

- **hint_text**: نص المساعدة (Text, محسوب)
  ```python
  def _compute_hint_text(self):
      for step in self:
          hints = []
          if step.step_type == 'manual':
              hints.append('هذه الخطوة تتطلب تدخل يدوي')
          if step.requires_approval:
              hints.append('تتطلب موافقة قبل المتابعة')
          if step.is_mandatory:
              hints.append('هذه الخطوة إلزامية')
          step.hint_text = '\n'.join(hints)
  ```
